# STANDARD_OPERATING_PROCEDURE

适用范围：`Codex`、`Claude Code` 及其他通用 Agent CLI

本文档定义团队在项目中实施 Agent token 优化时的标准操作流程。

## 1. 目的

本 SOP 用于统一团队在旧项目和新项目中实施 token 优化的动作顺序、交付物、验收标准与回退方式。

目标如下：

- 降低固定上下文成本
- 降低重复加载成本
- 不降低关键技能召回率
- 不明显增加澄清轮次、返工率和失败率

## 2. 适用对象

- 项目负责人
- 项目维护者
- 使用 Agent 进行开发、调试、研究或文档工作的成员

## 3. 角色分工

### 3.1 项目负责人

负责：

- 确认项目目录边界
- 确认优化目标和范围
- 审批重大上下文结构调整
- 评估体验是否退化

### 3.2 执行者

负责：

- 进行 token 成本审计
- 生成和维护项目文档
- 调整项目上下文结构和技能分层
- 记录结果与回退信息

### 3.3 审核者

负责：

- 复核实施方案
- 复核文档完整性
- 复核是否满足质量指标
- 复核是否引入新的体验风险

## 4. 项目启动前的统一动作

每次开始前，必须先确认以下事项：

1. 当前会话只服务当前项目目录
2. 当前项目目录是唯一默认项目根目录
3. 不默认读取其他兄弟目录
4. 明确这是“旧项目优化”还是“新项目初始化”

## 5. 旧项目标准流程

### Step 1: 确认项目根目录

- 确认当前项目路径
- 确认默认不跨目录读取
- 记录本次优化范围

### Step 2: 审计当前 token 成本来源

识别以下四类成本：

- 固定上下文成本
- 重复加载成本
- 技能暴露成本
- 历史携带成本

### Step 3: 生成项目基础文档

必须生成或更新：

- `PROJECT_CONTEXT.md`
- `PROJECT_SKILLS.md`
- `docs/token-optimization/token-optimization-report.md`
- `docs/token-optimization/token-optimization-checklist.md`

如项目已有会话管理机制，建议补充：

- `SESSION_SUMMARY_TEMPLATE.md`

### Step 4: 提出实施优先级

按以下顺序给出优化动作：

1. 缩短全局常驻层
2. 缩短项目默认上下文
3. 收缩技能默认暴露范围
4. 建立摘要机制
5. 建立新 skill 动态召回机制

### Step 5: 执行低风险优化

优先进行低风险改造：

- 瘦身全局说明
- 建立项目最小上下文
- 技能分层
- 历史摘要化

### Step 6: 验收

同时检查：

- token 是否下降
- 首轮技能命中率是否保持
- 首轮澄清次数是否保持
- 任务成功率是否保持

### Step 7: 记录回退点

每次做上下文结构调整后，必须记录：

- 调整内容
- 调整前状态
- 调整后状态
- 回退方法

## 6. 新项目标准流程

### Step 1: 先设计，不先堆规则

新项目启动阶段，先定义：

- 全局最小上下文
- 项目默认上下文
- 技能分层机制
- 摘要机制
- 动态召回机制

### Step 2: 初始化项目文档

必须创建：

- `PROJECT_CONTEXT.md`
- `PROJECT_SKILLS.md`
- `SESSION_SUMMARY_TEMPLATE.md`
- `docs/token-optimization/token-optimization-report.md`
- `docs/token-optimization/token-optimization-checklist.md`

### Step 3: 定义默认技能画像

至少分三层：

- 基础设施技能
- 项目默认技能
- 长尾冷层技能

### Step 4: 定义动态 skill 召回规则

至少覆盖以下信号：

- 新技术栈
- 新文件类型
- 新工作流
- 方案连续失败
- 任务模式升级

### Step 5: 验证是否轻量启动

验证要点：

- 简单任务是否快速进入正题
- 是否避免默认加载过长规则
- 是否避免默认暴露全部技能正文

## 7. 通用技能管理 SOP

### 7.1 基础设施技能

以下类型默认常驻索引：

- debugging
- tdd
- review
- planning
- documentation

### 7.2 项目默认技能

项目默认技能应控制在高频、小集合范围内，建议 5 到 15 个。

### 7.3 长尾技能

长尾技能默认不进入上下文，只在命中时按需加载。

### 7.4 新 skill 晋升流程

1. 第一次命中：只在当前任务使用
2. 短期重复命中：加入项目默认技能索引
3. 长期高频命中：晋升为项目常驻能力

## 8. 会话摘要 SOP

每次会话结束前，建议沉淀一份摘要，至少包含：

- 目标
- 当前状态
- 关键约束
- 已确认决策
- 未决问题
- 下一步

禁止默认把长会话原文直接作为下轮上下文。

## 9. 验收标准

### 成本验收

- 首轮固定上下文 token 下降
- 前 5 轮累计 token 下降
- 简单任务平均成本下降

### 质量验收

- 首轮技能命中率不下降
- 首轮澄清次数不显著上升
- 任务成功率不下降
- 返工率不显著上升

## 10. 失败判定

出现以下任一情况，应判定为失败优化：

- token 下降但体验明显变差
- 关键技能召回明显下降
- 简单任务明显变慢
- 返工率显著上升
- 团队成员需要频繁手动补上下文才能完成任务

## 11. 回退 SOP

若优化失败，按以下顺序回退：

1. 恢复上一个项目默认技能画像
2. 恢复上一版项目上下文文件
3. 恢复上一版摘要机制或历史携带方式
4. 恢复上一版常驻索引范围

回退后必须重新记录：

- 失败原因
- 受影响任务
- 临时补救措施
- 后续再尝试条件

## 12. 最小执行版本

若团队当前时间有限，只需先执行以下五步：

1. 确认项目根目录边界
2. 创建 `PROJECT_CONTEXT.md`
3. 创建 `PROJECT_SKILLS.md`
4. 创建 token 优化报告和清单
5. 建立 session summary 模板

这是最低成本且最容易落地的一版 SOP。
