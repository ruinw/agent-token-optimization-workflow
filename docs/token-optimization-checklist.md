# Agent Token 消耗优化实施清单

适用范围：`Codex`、`Claude Code` 及其他通用 Agent CLI

## 1. 基线测量

- [ ] 记录当前首轮固定上下文 token
- [ ] 记录前 5 轮累计 token
- [ ] 记录首轮澄清次数
- [ ] 记录任务完成率
- [ ] 记录返工率或失败率

## 2. 全局常驻层瘦身

- [ ] 识别当前全局常驻的系统提示和 AGENTS 文件
- [ ] 拆分长协议文件为入口摘要和阶段文档
- [ ] 确保全局常驻层只保留最小原则和边界
- [ ] 移除默认常驻的完整命令说明和完整阶段细节

## 3. 技能分层

- [ ] 列出基础设施技能
- [ ] 列出当前项目高频技能
- [ ] 列出长尾低频技能
- [ ] 保证基础设施技能始终可见
- [ ] 将长尾技能转为默认冷层

## 4. 技能发现与加载

- [ ] 建立轻量 skill 索引，仅包含名称、描述、关键词
- [ ] 确保 skill 正文只在命中时读取
- [ ] 确保用户显式点名 skill 时可以直接加载
- [ ] 确保高风险任务能优先召回基础设施技能

## 5. 项目级最小上下文

- [ ] 为项目创建极短 `PROJECT_CONTEXT.md` 或同类文件
- [ ] 只保留项目目标、技术栈、核心目录、禁区和常用命令
- [ ] 避免把长 README、长 KB、长历史直接设为默认上下文

## 6. 会话模式化

- [ ] 定义 `Lite` 模式
- [ ] 定义 `Dev` 模式
- [ ] 定义 `Research` 模式
- [ ] 为三种模式分别定义默认加载内容
- [ ] 验证轻任务不会误入重模式

## 7. 历史压缩

- [ ] 建立统一的 session summary 模板
- [ ] 默认只保留目标、状态、约束、决策、未决问题
- [ ] 阻止长会话原文直接续带
- [ ] 为摘要建立失效或重建规则

## 8. 项目演化中的新 skill 动态召回

- [ ] 定义“新领域信号”触发条件
- [ ] 新技术栈出现时执行一次全局 skill 搜索
- [ ] 新文件类型出现时执行一次全局 skill 搜索
- [ ] 当前方案连续失败时触发补充 skill 发现
- [ ] 将重复命中的新 skill 晋升为项目画像的一部分
- [ ] 用 `PROJECT_SKILLS.md` 或 `skills.enabled.json` 记录项目技能画像

## 9. 风险护栏

- [ ] 不削弱调试、TDD、评审、规划等基础设施能力
- [ ] 不显著增加首轮澄清次数
- [ ] 不显著降低首轮技能命中率
- [ ] 不让简单任务响应速度变慢
- [ ] 为所有优化措施准备快速回退方案

## 10. 验收

- [ ] 首轮固定上下文 token 下降
- [ ] 前 5 轮累计 token 下降
- [ ] 关键任务成功率不下降
- [ ] 首轮命中率不下降
- [ ] 返工率不显著上升
- [ ] 使用者主观体验未明显变差

## 11. 推荐执行顺序

1. 先测量基线
2. 先缩短全局常驻层
3. 再做技能分层
4. 再做历史摘要化
5. 最后做动态召回和更激进的按需加载
